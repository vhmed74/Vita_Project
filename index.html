Introduction &
Problem Statement

The VITA Solution

  • The VITA Objective: A single integrated portable platform.
  • Continuous, real-time multi-parameter tracking (no missed events).
  • Intelligent, automated signal interpretation replacing manual observation.
  • Meaningful alarms with recommended clinical actions.
  • The system integrates biomedical sensors with embedded electronics.
  • Multi-tier architecture: Acquisition, Processing, and Display.
  • Signal conditioning and preprocessing happen in real-time.
  • VITA guarantees continuous monitoring with no data loss.

System Architecture

Core Processor

  • Raspberry Pi 4 Model B serves as the central brain.
  • Manages the local database for storing patient history.
  • Hosts the GUI and runs the Python backend algorithms.
  • Capable of future cloud integration and remote access.
  • Arduino Nano acts as the data acquisition module (ADC).
  • Converts analog signals from sensors into digital data.
  • Ensures reliable serial communication with the Raspberry Pi.
  • Provides precise timing for continuous sensor sampling.

Data Acquisition

Pulse Oximetry

  • MAX30102 Sensor for Heart Rate and SpO2.
  • Uses Photoplethysmography (PPG) optical technology.
  • Non-invasive, highly accurate, and compact design.
  • Enables early detection of hypoxia or arrhythmias.
  • DS18B20 Digital Temperature Sensor.
  • High precision medical-grade thermal measurement.
  • Triggers alerts for fever or hypothermia conditions.
  • One-Wire interface ensures simple wiring to the MCU.

Body Temperature

Electrocardiogram (ECG)

  • AD8232 Single-Lead ECG Front End.
  • Extracts and amplifies small biopotential signals.
  • Built-in noise rejection for clear waveform monitoring.
  • Crucial for diagnosing cardiac anomalies in real-time.

Medical Algorithm

  • VITA goes beyond simple measurements by deriving additional vitals.
  • Blood Pressure (BP) & Respiratory Rate (RR): Estimated using mathematical models based on HR, SpO2, and Temp.
  • Reduces the number of physical sensors attached to the patient.
  • Smart decision algorithm compares data with clinical thresholds.

Planning & Methodology

Calibrated sensors against commercial medical devices to ensure clinical accuracy before beginning software integration.

Engineering Challenges

  • ECG Noise: High electrical interference on AD8232.
    Solution: Implemented hardware grounding & digital moving-average filters.
  • Serial Latency: Syncing 3 distinct sensors simultaneously.
    Solution: Optimized baud rate (9600) and structured packet formatting.

sensor_reader.py 
# VITA - Sensor Reading Module
import serial, threading
from collections import deque

class VitaSensorReader:
    def __init__(self, port='/dev/ttyUSB0', baud=9600):
        self.ser = serial.Serial(port, baud, timeout=1)
        self.ecg = deque(maxlen=500)
        self.hr = 0; self.spo2 = 0; self.temp = 0.0
        self.running = True

    def read_loop(self):
        while self.running:
            try:
                parts = self.ser.readline().decode('utf-8').strip().split(',')
                self.hr   = int(parts[0])   # AD8232 ECG
                self.spo2 = int(parts[1])   # MAX30102
                self.temp = float(parts[2]) # DS18B20
                self.ecg.append(float(parts[3]))
            except: pass
alert_engine.py 
# VITA - Clinical Alert Engine
THRESHOLDS = {'hr_high':100, 'hr_low':50, 'spo2_low':92, 'temp_high':38.5}

def evaluate(hr, spo2, temp):
    alerts = []
    if hr > THRESHOLDS['hr_high']:
        alerts.append(('CRITICAL', 'Tachycardia'))
    elif hr < THRESHOLDS['hr_low']:
        alerts.append(('WARNING',  'Bradycardia'))
    if spo2 < THRESHOLDS['spo2_low']:
        alerts.append(('CRITICAL', 'Hypoxemia'))
    if temp > THRESHOLDS['temp_high']:
        alerts.append(('WARNING',  'Pyrexia'))
    return alerts or [('NORMAL', 'Stable')]
